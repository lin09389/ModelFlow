## 🎯 核心功能

### 1. 智能分类
- 关键词匹配（高/中/低复杂度）
- 长度判断（字数/字符数）
- 任务类型识别（代码/写作/分析/聊天）
- 特殊模式检测（代码块/多步骤/推理）

### 2. 路由策略
- **balanced**（平衡模式）- 默认
- **cost-optimal**（成本优先）- 能降级就降级
- **performance**（性能优先）- 能升级就升级

### 3. 自动规则
- 自动升级（长文本/高复杂度）
- 自动降级（预算控制）
- 关键词覆盖（用户强制指定）

### 4. 成本追踪
- 实时统计各模型使用次数
- 预估每次请求成本
- 每日预算监控

## 🔧 集成方式

### 方案 A：手动调用（推荐）
```bash
RESULT=$(node integrate.js test "你的消息")
MODEL=$(echo $RESULT | grep modelRef | cut -d'"' -f4)
openclaw agent --message "你的消息" --model "$MODEL"
```

### 方案 B：源码集成（高级）
在 OpenClaw 会话处理流程中调用 `routeMessage()` 函数。

### 方案 C：环境变量（自动化）
```bash
export HYBRID_MODEL_ENABLED=true
```

## ⚠️ 注意事项

1. **模型引用格式**：确保配置的 `ref` 格式正确（如 `qwen/qwen-max`）
2. **成本估算**：基于输入 token，实际成本可能因输出长度而异
3. **分类准确性**：复杂任务可能需要手动调整关键词规则
4. **依赖检查**：运行前确保已安装 Node.js 和 jq
