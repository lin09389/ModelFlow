# Model Router - 多模型分层路由

## 核心目标

**用合适的模型做合适的事**，在不降低质量的前提下最小化成本。

---

## 模型分层

| 层级 | 模型 | 适用场景 | 相对成本 |
|------|------|----------|----------|
| **轻量** | `qwen-turbo` | 简单问答、格式转换、数据提取 | 💰 |
| **标准** | `qwen-plus` | 文档总结、代码审查、数据分析 | 💰💰 |
| **强** | `qwen-max` | 架构设计、创意写作、多步推理 | 💰💰💰 |

---

## 任务复杂度评估

### 自动判断规则

**轻量任务（qwen-turbo）**：
- [ ] 单步任务
- [ ] 不需要推理
- [ ] 有明确规则
- [ ] 示例：查天气、格式转换、简单问答

**中等任务（qwen-plus）**：
- [ ] 2-3 步
- [ ] 需要一定推理
- [ ] 涉及多个文件
- [ ] 示例：文档总结、代码审查、数据分析

**复杂任务（qwen-max）**：
- [ ] >3 步
- [ ] 需要创造性
- [ ] 涉及架构/设计
- [ ] 示例：系统设计、创意写作、策略规划

### 关键词检测

```yaml
# 轻量任务关键词
"简单"、"快速"、"转换"、"提取"、"格式化"

# 中等任务关键词
"分析"、"总结"、"审查"、"对比"、"优化"

# 复杂任务关键词
"设计"、"架构"、"规划"、"创意"、"策略"、"从零开始"
```

## 使用方法

### 自动路由（推荐）

```bash
# 直接说任务，自动选择模型
"帮我分析一下这个日志文件"
→ 自动判断为中等任务 → qwen-plus

"设计一个电商系统架构"
→ 自动判断为复杂任务 → qwen-max

"把这 10 个 JSON 转成 YAML"
→ 自动判断为轻量任务 → qwen-turbo
```

### 手动指定

```bash
# 明确指定模型
sessions_spawn model="qwen-turbo" task="简单任务"
sessions_spawn model="qwen-plus" task="中等任务"
sessions_spawn model="qwen-max" task="复杂任务"
```

---

## 最佳实践

### 1. 默认用标准模型

不确定时，用 `qwen-plus` 作为默认选择。

### 2. 简单任务果断用轻量

格式转换、数据提取等重复性工作，不要浪费强模型。

### 3. 复杂任务不要省

架构设计、创意写作等需要推理的任务，强模型一次搞定比重试更省钱。

### 4. 批量任务用并行

```bash
# 10 个独立任务，用 10 个轻量模型并行
同时 spawn 10 个 qwen-turbo
```
